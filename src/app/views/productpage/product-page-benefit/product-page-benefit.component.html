Kubera Propel is Fast. Really Fast

Kubera Propel is a cloud-native declarative data plane written in Rust. It’s built on the open source OpenEBS Mayastor. Kubera Propel combines breakthrough technologies like NVMe, SPDK, and emerging new storage features in the Linux kernel.

OpenEBS managed by Kubera Propel has been shown independent benchmarks to deliver extremely low latency performance for data bases and other workloads running on Kubernetes.

Key Benefits

  • 01Low-latency and NVMe support
  • 02Simple GUI with check and auto-configuration
  • 03CSI control plane

Run Anything, Anywhere

Wherever your code goes, Kubera follows. Kubera Propel is tested and validated with all major Kubernetes platforms. Runs on major clouds (EKS, AKS, GKE), and in popular k8s distributions like Red Hat Openshift, VMware Tanzu, and Rancher. Bring performance and cloud like convenience on-prem, to your hybrid cloud, or to a multi-cloud strategy.

Kubera Propel complements any modern data workload. From Kafka to Postgres, Kubera Propel is a great match for all of the most popular databases and stateful workloads.

Packed with Features

Kubera Propel has all of the enterprise features needed to orchestrate storage at scale. It comes packed with all of the policy, partitioning, protection, and performance benchmarking features you would expect.

Kubera Propel is fast, but it’s also featureful. Many of the essential features are enabled out of the box, and adding options requires just a flip of a switch.
